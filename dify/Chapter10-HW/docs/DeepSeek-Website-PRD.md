# SeekDeep 官网项目产品需求文档 (PRD)

## 1. 项目概述

* **项目名称**：SeekDeep 官方网站建设
* **产品定位**：高性能 AI 技术门户，提供沉浸式 AI 对话体验及开发者生态服务。
* **核心理念**：极简、极速、极智。

---

## 2. 页面结构与功能分析

DeepSeek 的官网由三个核心板块组成：**Chat 交互区（核心）**、**开发者平台（API）**、**技术文档/企业信息**。

### 2.1 核心交互区 (Home/Chat)

这是仿照 DeepSeek 的重头戏，采用“即开即用”的对话界面。

* **沉浸式对话框**：页面中心为输入框，支持长文本输入，具备自动扩容功能。
* **模型切换开关**：支持“标准模式”与“深度思考（Reasoning）”模式切换（仿 DeepSeek-R1 的逻辑显示）。
* **侧边栏管理**：
* 历史记录：按时间排序的对话列表。
* 新对话按钮：置顶明显位置。


* **搜索增强开关**：集成联网搜索插件开关。

### 2.2 开发者开放平台 (Platform)

* **API 密钥管理**：用户可自助创建、查看、禁用 API Key。
* **用量统计图表**：展示 Token 消耗、并发量等实时数据。
* **充值与账单**：支持轻量级的额度充值体系。

### 2.3 静态信息区 (Footer/Nav)

* **开源模型链接**：指向 GitHub 仓库。
* **技术白皮书**：PDF 在线预览或下载。
* **公司/品牌信息**：关于我们、隐私政策、服务协议。

---

## 3. UI/UX 视觉规范

SeekDeep 应严格参考 DeepSeek 的**工业设计感**：

| 维度 | 设计要求 |
| --- | --- |
| **配色方案** | 背景以 `#FFFFFF` (亮色) 和 `#000000` (深色) 为主，辅助色采用科技蓝或深空灰。 |
| **字体** | 优先使用无衬线字体（如 Inter, PingFang SC），强调阅读的清晰度。 |
| **交互动画** | 拒绝多余动效。仅保留对话打字机效果、侧边栏滑动以及 Tab 切换的微交互。 |
| **响应式** | 必须适配移动端，Chat 界面在手机端需保持输入框固定在底部，上方滚动对话。 |

---

## 4. 关键功能需求 (Feature List)

### F01：深度思考过程可视化

* **需求描述**：当用户开启“深度思考”模式，AI 返回结果前需展示 `思考中...` 过程折叠框。
* **验收标准**：点击可展开完整的 Chain-of-Thought（逻辑链路），再次点击可折叠。

### F02：Markdown 高级渲染

* **需求描述**：代码块需支持语法高亮、一键复制；数学公式需支持 LaTeX 渲染。
* **验收标准**：渲染速度无延迟，支持表格、多级标题和加粗等格式。

### F03：流式输出 (SSE)

* **需求描述**：采用 Server-Sent Events 技术，实现逐字排版输出。
* **验收标准**：响应首包延迟（TTFB）应控制在 500ms 以内。

---

## 5. 技术架构建议 (Tech Stack)

为了达到 DeepSeek 官网那种丝滑的体验，建议采用以下组合：

* **前端框架**：**Next.js 14/15** (支持 SSR 优化 SEO，同时对 Chat 场景的流式响应支持极好)。
* **样式库**：**Tailwind CSS** (实现极简 UI 的最快方式)。
* **状态管理**：**Zustand** 或 **TanStack Query** (处理对话流与 API 状态)。
* **后端支撑**：**Python (FastAPI)** 或 **Node.js**，配合 **Redis** 处理对话上下文缓存。

---

## 6. 非功能性需求

1. **加载性能**：首屏加载时间（LCP）小于 1.5 秒。
2. **安全性**：API 访问必须经过严格的鉴权，防范提示词攻击（Prompt Injection）。
3. **兼容性**：主流浏览器（Chrome, Safari, Edge）及手机端浏览器 100% 兼容。
